객체 지향 4대 특징 (캡슐화, 상속, 다형성, 추상화)에 대해서 공부한 후 정리

캡슐화
데이터를 외부에서 함부로 접근하지 못하도록 보호하는 것
접근 제한자 private를 통해 정보 은닉을 하고 getter와 setter 메서드를 통해 접근하도록 한다.

상속
기존 클래스의 속성과 기능을 상속받아 새로운 클래스를 만드는 것
extends를 통해 부모 클래스를 상속하는 자식 클래스를 생성한다.

다형성
하나의 객체가 여러 형태를 가질 수 있는 것
부모 타입 하나로 여러 자식 객체를 관리할 수 있게 한다.
이때 오버라이딩을 통해 같은 명령이지만 객체에 따라 다른 행동을 하게 할 수 있다.

추상화
불필요한 특성은 제거하고 핵심 기능만 남기는 것
추상클래스와 인터페이스로 활용되는 개념이다.
추상클래스는 extends로 상속해 사용하며 
인터페이스는 implements를 통해 사용되는데 추상클래스와 다르게 다중 상속이 가능하다.
